# Marketing Templates (Phase9/10 ドラフト)

目的: 実装を増やさず、ローンチ初期に使える最低限のマーケ素材と運用の「型」を準備する。キャラクターや課金ロジックの改変は行わない。

## 価値提案 1 文（T10-01）
「忙しい日でも、3 タップで今の気持ちを整えて次の一歩が見えるタロットボットです。」

## 投稿テンプレ（週7本の型：T10-03）
1. **悩みの提示**: 「予定が詰まって心が落ち着かないときは？」
2. **共感フレーズ**: 「わかります、そのモヤモヤを抱えたまま動くのはしんどいですよね。」
3. **小さな提案**: 「まずは状況を一言で教えてください。カードと一緒に整理してみましょう。」
4. **導線**: 「/start から無料で試せます。もっと深く知りたいときは /buy でチケット追加もできます。」

### バリエーション例
- **恋愛**: 「あの人との距離感がわからないときに、今できる一歩を一緒に考えます。」
- **仕事**: 「転職か続投か迷うとき、整理のためのヒントを 3 枚のカードで。」
- **人生/自己理解**: 「自分のペースを取り戻したいときに、今週のキーワードをカードから受け取る。」

## KPI 最小セット（T10-05）
- **開始率**: `/start` 実行ユーザー数 ÷ 新規流入数（SNS クリック数）。
- **完了率**: `/start` 後に占い結果を受け取ったユーザー ÷ `/start` 実行ユーザー。
- **課金率**: `/buy` 決済完了ユーザー ÷ 占い完了ユーザー。
- **継続率**: 初回占いから 7 日後に再訪したユーザー ÷ 初回ユーザー。
- **粗利**: Stars 売上合計 − OpenAI 利用料（手動集計）。

## 計測と確認手順
- `/admin stats 7` で占い/相談/決済/エラー件数を日次で確認（直近 14 日まで）。離脱点は「占い完了 < `/start` 数」「決済 < 購入導線クリック数」を見る。
- OpenAI Usage から日次トークン使用量を CSV で取得し、粗利計算に足す。閾値超過が近いときは `THROTTLE_*` で負荷を抑えるか案内文を優しめにする。
- SNS 投稿の URL には UTM（例: `?utm_source=x&utm_medium=social&utm_campaign=launch`）を付け、クリック数はプラットフォームの簡易分析で確認する。
